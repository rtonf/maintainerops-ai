# v0.1.10 Model Eval and Trusted Publishing Release Candidate

MaintainerOps AI `v0.1.10` is the release candidate for model-backed eval hardening and npm Trusted Publishing.

## Highlights

- Bumps the source package version to `0.1.10`.
- Adds `.github/workflows/npm-publish.yml` for npm Trusted Publishing from a published GitHub Release or manual tag dispatch.
- Keeps npm publishing tokenless: the workflow relies on npm's Trusted Publisher OIDC path and does not use `NPM_TOKEN`.
- Expands the selected manual model-backed eval set from 2 to 5 cases.
- Keeps live model evals manual-only and budget-gated with `--budget-usd`, `--max-cases`, and `--max-output-tokens`.
- Normalizes model-suggested labels to canonical project labels before strict eval comparison.
- Adds work-item-aware safeguards for release-readiness issues and actionable prompt-injection/security issues.

## Verification Plan

- `npm run verify`
- `npm run eval:model -- --budget-usd 0.5 --max-cases 5 --max-output-tokens 1200`
- Pull request checks, including CodeQL on merge.

- Clean npm install proof after publication.
