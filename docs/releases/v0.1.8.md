# v0.1.8 Security Remediation Release Candidate

Date: 2026-06-24

## Summary

This patch release publishes the fixes validated by the 2026-06-23 repository-wide Codex Security rescan. It strengthens secret handling, bounds unusually large pull-request inputs, and hardens the repository's read-only GitHub Actions workflow.

## Security and Reliability Changes

- Redact the complete serialized model payload before applying lossy truncation.
- Remove duplicate aggregate PR patch storage when per-file patches are available.
- Limit offline analyzer search input to 1,000,000 characters before joining and lowercasing.
- Preserve actionable missing-auth and permission findings when issue text also requests feedback.
- Run pull-request analysis with `pull_request_target`, read-only permissions, no contributor checkout, and a commit-pinned published Action.
